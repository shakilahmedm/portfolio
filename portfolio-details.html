Ask--Your--Doc-

This project is a Retrieval-Augmented Generation (RAG) API and web application that allows users to upload documents and ask questions based on their content. It supports multiple file types including PDF, DOCX, TXT, images (OCR), CSV, and SQLite databases. The system extracts text, generates embeddings using Google Gemini, stores them in a FAISS vector database, and retrieves relevant context to answer user queries accurately. The backend is built with FastAPI, the frontend uses Streamlit, and the entire application can be run locally or deployed using Docker. Users must upload a document before asking questions; otherwise, answers are unavailable due to missing context. Caching is used for performance, and refreshing the page ensures updated document indexing.

Key Features
  • Upload and chat with multiple document types
  • OCR support for images and scanned PDFs
  • Fast semantic search using FAISS
  • Multi-document querying
  • Minimal Streamlit frontend
  • Fully containerized with Docker
  • You will get full question answer metadata
Technologies Used
  • Python 3.9+
  • FastAPI (Backend)
  • Streamlit (Frontend)
  • FAISS (Vector Search)
  • LangChain
  • Google Gemini LLM

IMDB Movie Review Classification

This project is a Sentiment Analysis application that classifies movie reviews as positive or negative using machine learning techniques. It processes raw text data through cleaning and preprocessing, converts text into numerical features using TF-IDF, and applies supervised classification models to predict sentiment accurately. The project is implemented in Python using Jupyter Notebook (Google Colab recommended) and evaluates multiple models including Logistic Regression, Naive Bayes, Random Forest, Decision Tree, Gradient Boosting, and Support Vector Machine (SVM). Model performance is measured using standard classification metrics, and the best-performing model is saved for reuse and real-time prediction.

Key Features
  • Text preprocessing (HTML removal, stopwords, normalization)
  • Feature extraction using TF-IDF (Unigrams/Bigrams)
  • Multiple ML model comparison (SVM, RF, Logistic Reg)
  • Evaluation via Accuracy, Precision, Recall, and F1-score
  • Confusion matrix visualization
  • Model persistence (saved as .pkl)
  • Interactive demo for real-time sentiment prediction

Salary-Prediction-Classification

A neural network was developed using TensorFlow and Keras to predict if an individual earns over $50K annually. Trained on census data (cleaned and preprocessed), the model achieved a test accuracy of approximately 86.1% and a training accuracy of around 87.6%. The training process involved 300 epochs and resulted in a final test loss of about 0.32 and a training loss of roughly 0.26.

Telco-Customer-Churn-Analysis

The Telco Customer Churn Analysis project is a comprehensive examination of customer attrition within a telecommunications dataset. The primary objective is to identify the demographic patterns and service-related drivers—such as tenure, contract type, and payment methods—that lead customers to discontinue their services. By uncovering these patterns, the project provides actionable, evidence-based strategic recommendations to improve customer retention and reduce turnover.

Key Features
  • Comprehensive Data Cleaning: Processing raw data by handling blank spaces in financial columns and converting data types for accurate analysis.
  • Demographic & Behavioral Analysis: Comparing churn rates across different segments, including senior citizens, gender, and customer loyalty (tenure).
  • Service Impact Evaluation: Analyzing how specific services, such as Fiber Optic internet or Online Security, correlate with customer retention.
  • Financial & Payment Insights: Investigating the influence of monthly charges and payment methods (e.g., electronic checks vs. auto-pay) on churn behavior.
  • Advanced Visualizations: Utilizing heatmaps, swarm plots, and grouped bar plots to illustrate correlation patterns and trends
